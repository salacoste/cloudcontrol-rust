## Architecture Evolution

Each story added a control channel building on the AtxClient foundation:

```
3-1: Tap                ->  AtxClient.click(x,y) via JSON-RPC + REST POST /control/{udid}/tap
3-2: Swipe              ->  AtxClient.swipe() + ADB fallback for MIUI compatibility
3-3: Text Input          ->  AtxClient.input_text() with FastInputIME + ADB fallback
3-4: Key Events          ->  AtxClient.press_key() + key name-to-KEYCODE mapping in Adb
3-5: UI Hierarchy        ->  AtxClient.dump_hierarchy() + quick-xml -> JSON tree parser
3-6: Shell Command       ->  AtxClient.shell_cmd() via GET /shell?command= + Adb.shell()
```

## Successes

1. **ATX-first with ADB fallback** -- Every control operation tries the ATX JSON-RPC path first (faster, works over WiFi), then falls back to ADB shell commands (universal, works with USB). The NIO WebSocket handler applies this pattern consistently across all event types.
2. **MIUI swipe workaround** -- ATX JSON-RPC `swipe` silently fails on Xiaomi/MIUI devices. The NIO handler detects when a serial is available and routes swipes directly through `Adb::input_swipe()` which uses the `input swipe` shell command that works universally.
3. **Key name normalization** -- `Adb::input_keyevent()` maps human-readable key names (home, back, enter, del) to Android KEYCODE constants. The NIO handler adds a second mapping layer for browser key names (Enter -> enter, Backspace -> del, Escape -> back), enabling direct keyboard passthrough from the web UI.
4. **XML hierarchy parser** -- `hierarchy::xml_to_json()` converts Android UIAutomator XML to a JSON tree with attribute aliasing (class -> className, resource-id -> resourceId, content-desc -> description, bounds -> rect with computed width/height). Each node gets a UUID for frontend tree rendering.
5. **WebSocket command multiplexing** -- The NIO handler multiplexes screenshot streaming, touch, swipe, input, and keyevent over a single WebSocket connection per device. The `{type, data}` message protocol keeps the client implementation simple.

## Challenges

1. **Text input escaping** -- ADB shell `input text` requires escaping spaces as `%s`, backslashes as `\\\\`, and double quotes as `\\\"`. The ATX `sendKeys` path via FastInputIME avoids this complexity entirely, which is why it's the preferred path.
2. **Coordinate scaling** -- The web UI renders screenshots at reduced scale (0.5x). Touch coordinates from browser click events must be scaled back to device resolution before sending to ATX/ADB. This scaling is handled client-side in remote.js, not server-side.
3. **UI hierarchy size** -- Full UIAutomator XML dumps can be 500KB+ for complex apps. The quick-xml streaming parser handles this efficiently without loading the entire XML into memory, but the resulting JSON tree can still be large for deeply nested views.
4. **Shell command timeout** -- `AtxClient.shell_cmd()` uses the default 15s HTTP timeout. Long-running shell commands (e.g., package install) can time out. ADB shell has no timeout by default, creating an inconsistency between the two paths.

## Technical Debt

| Item | Severity | Description |
|------|----------|-------------|
| Swipe MIUI workaround hardcoded | Medium | All swipes with a serial go through ADB. Should detect MIUI specifically and use ATX for other ROMs. |
| No shell command timeout control | Low | Shell commands via ATX use fixed 15s timeout. Should be configurable per-request. |
| Key validation incomplete | Low | `Adb::input_keyevent()` passes unknown keys directly to `input keyevent`. Invalid keys silently fail. |
| No long-press support | Low | Only tap (instant click) is supported. Long-press requires a swipe with zero distance and long duration, which is a workaround. |

## Action Items

1. Add MIUI detection to route swipes through ADB only when necessary
2. Add configurable timeout parameter for shell command execution
3. Implement dedicated long-press endpoint (or detect zero-distance swipe as long-press)
