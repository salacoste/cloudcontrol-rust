## Successes

1. **Existing infrastructure reuse** — Stories 10.2 and 10.3 had significant head starts:
   - 10.2: Route, DB column, frontend code ALL already existed — only handler logic needed
   - 10.3: All ACs already met — only diagnostic enhancements added

2. **Compile-time version extraction** — Story 10.1 used `env!("CARGO_PKG_NAME")` and `env!("CARGO_PKG_VERSION")` for zero-runtime-cost version info.

3. **Atomic reservation with DashMap entry API** — Story 10.2 code review caught a TOCTOU race condition. Fixed with `DashMap::entry()` API for atomic reservation.

4. **Thorough dead code verification** — Story 10.4 verified removed methods weren't referenced in any HTML templates before removal.

5. **Security hardening in upload** — Story 10.4 code review added:
   - 100MB upload size limit (OOM protection)
   - `Path::file_name()` sanitization (path traversal protection)

6. **Test coverage discipline** — Every story added integration tests. Total tests grew from 181 to 293 across Epic 10.

## Challenges

1. **WebSocket reservation conflict handling** — Story 10.2 needed to handle the case where frontend already attempted WebSocket upgrade before checking reservation. Solution: accept upgrade, send error JSON, then close.

2. **FPS counter freeze on disconnect** — Story 10.3 code review found FPS counter displayed stale value when streaming stopped. Fixed with 2-second timeout to reset to 0.

3. **Status bar state tracking** — Story 10.3 had misleading "Connected" text after disconnect. Required explicit `wsConnected` flag guard.

4. **Upload multipart handling** — Story 10.4 required careful multipart parsing with size limits and path sanitization.

## Patterns Discovered
