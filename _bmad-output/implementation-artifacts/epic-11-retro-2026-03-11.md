## Successes

1. **FFmpeg child process management** — Story 11.1 successfully implemented JPEG-to-MP4 encoding via FFmpeg stdin pipe, following the ScrcpyManager pattern from Epic 6.

2. **Graceful degradation without FFmpeg** — Server starts even if FFmpeg is unavailable; video endpoints return 503 with clear error message.

3. **Startup recovery mechanism** — Story 11.2 implemented comprehensive recovery:
   - Marks stale "recording" entries as "failed"
   - Registers orphaned MP4 files found on disk
   - Marks DB entries with missing files as "failed"

4. **Frontend restoration** — Story 11.1 restored recording methods that were removed as dead code in Story 10.4, now with server support.

5. **Dual-store pattern** — DashMap for active recordings (FFmpeg handles), SQLite for completed recordings. Clean separation of concerns.

6. **Tera/Vue conflict prevention** — Story 11.2 added `{% raw %}` blocks to remote.html for Vue template expressions.

## Challenges

1. **DashMap guard across await** — Story 11.1 code review caught holding DashMap reference across `.await` points. Fixed by cloning handles out before async operations.

2. **Video list not loading** — Story 11.2 code review found `resp.success` check in JavaScript — API returns `resp.status === 'success'`.

3. **N+1 query in recovery** — Story 11.2 code review caught `list_videos()` inside a loop. Fixed by moving query outside and using HashSet for lookups.

4. **Download for recovered videos** — Story 11.2 code review found download endpoint only allowed `status: "completed"`. Fixed to also allow `"recovered"`.

5. **TempDir cleanup race** — Story 11.2 code review found test database temp directory being dropped prematurely. Fixed by returning TempDir from test helper.

## Key Takeaways

1. **External dependencies need graceful degradation** — FFmpeg unavailability doesn't crash the server.

2. **Startup recovery is essential for file-backed systems** — Orphaned files and stale DB records must be reconciled.

3. **Dual-store patterns separate concerns cleanly** — DashMap for active, SQLite for persistent.

4. **Code review catches async ownership bugs** — DashMap guards across await points are a common Rust pitfall.

5. **Frontend-backend contracts matter** — `resp.success` vs `resp.status === 'success'` caused video list to not load.
